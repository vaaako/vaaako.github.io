FTP

Transfering files between computers over the network

Usage Example:

A website has an open port with this service (Usually port 22), where you can connect anonymously, (in nmap shows ftp-anon: Anonymous FTP login allowed (FTP code 230)) so you can connect via ftp (ftp <ip/site>) with the username "anonymous", then download the files from inside using get <filename>

SSH

Learn about this
Basically, remotely connects to a computer

Usage Example:

Basic: ssh <root>@<ip> With a private key: ssh -i id_rsa <root>@<ip>

  • The private key is in a file called id_rsa
  • Before run the command, run: chmod 600 id_rsa
  • If print some message with the options "yes/no/[fingerprint]", type yes

SCP

Tranfer files from your PC to the ssh connected system

Usage Example: scp <filename> <root>@<ip>:/dev/shm (Recommended always use this directory)

Useful Terminal Commands

Default terminal commands

Not listened the famous command, like mv, cp, ls, cd etc

|, && and ;

Pipe

Use it to the output of the first command be the input of the second command

Example: sort <filename> | tee output.txt

This command will sort the file lines (Before pipe) then save to a file named output.txt (After pipe)

&&

Multiple arguments to one command

Example sudo update && upgrade

Update and upgrade

;

Use it to execute multiple commands in the same line

Example: touch file.txt; cat file.txt

Create a file named file.txt then show the file content

Make .onion site

Install nginx and tor

sudo dnf install nginx tor


Run tor for the first time then add this things to torrc

tor
sudo echo "
HiddenServiceDir /var/lib/tor/hidden_service/
HiddenServicePort 80 127.0.0.1:80" >> /etc/tor/torrc # You can use nano or vim to go at line 78 & 79 and uncomment the two lines


Start nginx and tor (Run this when start a new setion)

sudo systemctl start nginx
sudo systemctl start tor


Show your url

sudo cat /var/lib/tor/hidden_service/hostname # Try use "sudo su" before this command


With that url go to Tor Browser and visit your site
Your site files: 

- `/etc/nginx/nginx.conf.` - Config file
- `/usr/share/nginx/html` - Root directory


On the main page are some useful things that is important to know

To apply changes, restart nginx: 

sudo systemctl restart nginx.service
